## Local Setup

Backend:

```bash
cd backend
pip install -r requirements.txt
python main.py
```

Frontend:

```bash
python -m http.server 8001 --directory frontend
```

Local URLs:

- Frontend: `http://127.0.0.1:8001`
- Backend: `http://127.0.0.1:8000`
- Swagger: `http://127.0.0.1:8000/docs`

## Android package for Google Play

The app is prepared as a PWA/TWA candidate.

Current Android values:

```text
App URL: https://seufuturo.blog.br
Manifest URL: https://seufuturo.blog.br/manifest.json
Package ID: br.com.hypersecit.seufuturo
Build script: scripts/build-android-twa.sh
```

To generate the Android package locally:

```bash
chmod +x scripts/build-android-twa.sh
./scripts/build-android-twa.sh
```

Expected output: an Android App Bundle file with `.aab` extension inside the generated build directory.

After generating the `.aab`:

1. Open Google Play Console.
2. Go to the app.
3. Go to Closed testing.
4. Create a new release.
5. Upload the `.aab` file.
6. Save and publish the closed test release.
7. Add at least 12 real testers.
8. Copy the official test link generated by Google Play.

The official closed-test link only exists after a release is created in Google Play Console.
